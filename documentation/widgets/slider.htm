Experis Slider Widget

Creates an interactive image or content slider from explicity defined HTML

Remarks

Like all functions in the Experis JS library, the slider widget is constantly evolving and may contain bugs and unexpected behavior when used in specific circumstances. The widget was initially designed strictly as an image slider, but has been modified to support any type of content and will even support multiple images per slide.

Options provided to the method allow you to determine the delay between slides, the type of transition (slide or fade), and whether the slider should scroll back to the beginning when it reaches the last slide or whether it should simply continue as if the first slide were next in the list.

The slider uses minimal CSS to accomplish its purpose, relying on the designer to provide the style.

Note: the slider next/prev arrows will be drawn outside of the slider's wrapper element. This allows the arrows to extend outside of the slider if necessary but also means that a secondary wrapper may be needed to contain the arrows. In the future, this will likely be revised so that the slider creates its own internal wrapper element.

Syntax

$xw.slider(el:dom element[, options:object]);

The slider widget expects a specific HTML syntax in order to perform correctly. The format consists of a wrapper element (any block-level element, such as a <div>) that contains a <div> element for each slide. An <ul> element can be optionally added within the wrapper to provide navigation.

<div id="slider">
	<!-- Begin optional navigation -->
	<ul>
		<li><a href="#slide-1">Slide One</a></li>
		<li><a href="#slide-2">Slide Two</a></li>
	</ul>
	<!-- End optional navigation -->

	<!-- A <div> element is required for each slide. The contents are unimportant,
	     but should be the same width and height as other slides. -->
	<div id="slide-1"><a href="url-1.htm"><img src="image-1.jpg" alt="" /></a></div>
	<div id="slide-2"><a href="url-2.htm"><img src="image-2.jpg" alt="" /></a></div>
</div>

Parameters

Parameter Default Value Available Values Description
el null dom element

This parameter is the “wrapper” element of the slider and is required. No error will be thrown if the element does not exist. This allows the script to be used on pages that may not have a slider without alerting the user of a problem.
options.delay 5000 (5 seconds) number

The amount of time the slider should pause between slides.
options.transTime 500 (½ second) number

The amount of time it takes for a slide to transition to the next.
options.transEffect 'slide' 'slide' or 'fade'

The type of effect that will be used to transition the slides.
options.finishEffect 'cycle' 'cycle' or 'none'

The 'cycle' setting will cause the slider to scroll back to the beginning when it reaches the end. The 'none' setting will cause the slider to continue moving forward with the first slide next in line at the end. This parameter has no effect when options.transEffect is set to 'fade'.
options.showArrows true boolean

Toggle whether the next/prev slide arrows are available.
options.width null number

The width of the slider (in pixels). If omitted, the width of the first image will be used.
options.height null number

The height of the slider (in pixels). If omitted, the height of the first image will be used.
options.imagesPerSlide 1 number

The number of slides to scroll after each transition. Make sure to set the options.width parameter if you intend to set this parameter to anything higher than 1.
