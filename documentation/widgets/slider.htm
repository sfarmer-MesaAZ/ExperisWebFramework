Remarks

Like all functions in the Experis JS library, the slider widget is constantly evolving and may contain bugs and unexpected behavior when used in specific circumstances. The widget was initially designed strictly as an image slider, but has been modified to support any type of content.

Options provided to the method allow you to determine the delay between slides, the type of transition, and other settings.

The slider uses minimal CSS to accomplish its purpose, relying on the designer to provide the style.

Syntax

$xw.slider(el:dom element[, options:object]);

The slider widget expects a specific HTML syntax in order to perform correctly. The format consists of a wrapper element (any block-level element, such as a <div>) that contains a <div> element for each slide. An <ul> element can be optionally added within the wrapper to provide navigation.

<div id="slider">
	<!-- Begin optional navigation -->
	<ul>
		<li><a href="#slide-1">Slide One</a></li>
		<li><a href="#slide-2">Slide Two</a></li>
	</ul>
	<!-- End optional navigation -->

	<!-- A <div> element is required for each slide. The contents are unimportant,
	     but should be the same width and height as other slides. -->
	<div id="slide-1"><a href="url-1.htm"><img src="image-1.jpg" alt="" /></a></div>
	<div id="slide-2"><a href="url-2.htm"><img src="image-2.jpg" alt="" /></a></div>
</div>

Important: if you are supporting IE 8 or below, do not use HTML 5 elements in the slides. The slider script dynamically creates new HTML content via cloneNode(), and the HTML 5 shim/shiv does not appear to support styling of the new content.

Parameters

Parameter Default Value Available Values Description
el null dom element

This parameter is the “wrapper” element of the slider and is required.
options.delay 5000 (5 seconds) number

The amount of time the slider should pause between slides.
options.transTime 300 (0.3 seconds) number

The amount of time it takes for a slide to transition to the next.
options.transType 'slide' 'slide', 'wipe', or 'fade'

The type of effect that will be used to transition between slides.
options.transEasing 'linear' 'linear', 'easeIn', 'easeOut', 'easeInOut', 'bounceIn', 'bounceOut'

Determines the transition’s easing effect. The “bounce” settings do not apply to “fade” transition, and the “bounceIn” setting does not apply to the “slide” transition.
options.showArrows true boolean

Toggle whether the next/prev slide arrows are available.
options.stopAfterHover true boolean

When set to true, the slide show will not automatically resume after the user has hovered over it
