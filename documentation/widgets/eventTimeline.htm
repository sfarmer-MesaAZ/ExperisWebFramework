Experis Event Timeline Widget

Creates a horizontal timeline of events with tooltip summaries

Details

The Event Timeline widget creates a timeline of events, where each event is marked by a colored dot at the appropriate location on the timeline. Hover over a dot, and a tooltip will appear. Click a dot to go to another URL (most likely an event detail page). Each event can be assigned to a specific category, and a legend will be auto-generated to color-code the events based on category.

Example

The example below uses the following options: 

{ months:3, monthWidth:300, sort:'descending', legendColors:{ News:'#900', Event:'#090', Press:'#009' } }

September 12, 2012

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer et enim magna, posuere congue lorem.

September 11, 2012

In elementum, nunc eu aliquet molestie.

August 22, 2012

Suspendisse est purus, laoreet et porttitor ac, rutrum sit amet ante. Nullam ut fermentum felis.

August 20, 2012

Donec egestas, metus at vestibulum iaculis, sapien turpis pharetra sem, id condimentum augue lacus ac mi.

August 20, 2012

Vivamus ligula dolor, mollis sed lacinia at, luctus non neque.

August 4, 2012

Nam convallis lacinia diam, at malesuada nulla scelerisque eget.

July 18, 2012

Vivamus ligula dolor, mollis sed lacinia at, luctus non neque.

Syntax

$xw.eventTimeline(wrapper:dom element[, options:object]);

The following HTML structure is required for the timeline:

<div id="wrapper">
	<div class="timeline-item" data-category="Category Name">
		<h2>January 1, 2012</h2>
		<p><a href="details1.htm">Tooltip content goes here</a></p>
	</div>

	<div class="timeline-item" data-category="Another Category">
		<h2>February 2, 2012</h2>
		<p><a href="details2.htm">Tooltip content goes here</a></p>
	</div>
</div>

Parameters

Parameter Default Value Available Values Description
wrapper null dom element

The “container” element for the timeline
options.months 4 number

The number of months to include on the timeline
options.monthWidth 200 number

The width (in px) of each month on the timeline
options.sort ascending string

Whether the timeline dates should start at the current month and ascend or start at the latest event date and descend. Possible values are "ascending" and "descending"
options.borderColor #000 string

The color value for the timeline’s border
options.legendColors {random: ['#f00', '#0f0', '#00f', '#ff0', '#0ff', '#f0f', '#f90', '#f09', '#0f9', '#09f', '#90f', '#9f0']} object

By default, the widget will assign a color chosen at random from the default array to each event category. To specify your own colors, provide an object similar to the following, where each category is represented:

{'Category Name': '#3f8ab2', 'Another': 'rgb(12, 120, 231)'}

Notes

  • IE 8 and below do not support the CSS3 border-radius property, so events appear as squares instead of circles. A background image could be used if circles are a requirement.
  • IE 7 has z-index issues with the event dots and tooltips. This can be fixed by adjusting the z-index values of the month divs manually.
