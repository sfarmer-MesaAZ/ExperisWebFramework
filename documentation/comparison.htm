So What’s Missing?

You may have noticed that unlike many other CSS frameworks, EWF does not provide icon sets, form styling, jQuery widgets, and other graphical assets. We feel that a framework should be just that — a framework. It should be a structural support, a building block, something that holds your website together — not something that dictates a website’s design or takes away from its uniqueness. A designer should be free to determine how forms should appear and what icon sets best match the look that is needed. If you’re looking for a shortcut where this type of work is already done for you, then you can find plenty of resources online already. We don’t feel that graphic design is the domain of a CSS framework.

Some responsive frameworks also include classes that allow you to hide or show content depending on the device (e.g. .hidden-phone or .show-on-desktops). Not only is this bad practice, as you are forcing users to download content they won’t even see, but it’s not a future proof solution. The screen resolution of phones and tablets is constantly increasing and can no longer be reliably determined by browser width alone. Many tablet computers already have desktop resolutions, so using a “dont-show-on-tablet” class isn’t going to work anymore. If you really want to hide content for particular screen widths, feel free to write the media queries yourself, but there is a reason we don’t include them in the framework.
